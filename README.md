# DoH HTTP Proxy  
  
**DoH HTTP Proxy** یک پروکسی HTTP محلی برای ویندوز است که نام دامنه‌ها را به‌جای DNS معمولی، از طریق **DNS over HTTPS (DoH)** resolve می‌کند. هدف برنامه این است که در شرایطی که پاسخ‌های DNS دستکاری، جعل، مسدود یا ناپایدار می‌شوند، اتصال  
کاربر تا حد امکان با استفاده از پاسخ‌های معتبر و cache شده برقرار بماند.  
  
در بسیاری از اختلال‌های مبتنی بر DNS، هنگام تبدیل نام دامنه به IP، پاسخ نادرست برگردانده می‌شود یا درخواست DNS اصلاً به  
مقصد نمی‌رسد. در چنین وضعیتی، مرورگر یا برنامه‌ی کاربر نمی‌تواند مقصد واقعی را پیدا کند، حتی اگر خودِ مسیر اتصال به  
اینترنت هنوز برقرار باشد.  
  
این برنامه با ایجاد یک پروکسی محلی، درخواست‌های HTTP/HTTPS را دریافت می‌کند، دامنه‌ی مقصد را از طریق endpointهای DoH  
resolve می‌کند و سپس اتصال را بر اساس IP به‌دست‌آمده برقرار می‌سازد. علاوه بر این، می‌تواند نتایج resolve شده را ذخیره  
کند، از چند DoH endpoint به‌صورت fallback استفاده کند، IPهای بهتر را برای اتصال انتخاب کند و در حالت اضطراری، ابتدا از  
داده‌های قبلی یا فایل `hosts` کمک بگیرد.  
  
این ابزار برای شرایطی مناسب است که اتصال مستقیم یا غیرمستقیم به اینترنت همچنان ممکن است، اما DNS شبکه قابل‌اعتماد نیست.  
در مقابل، اگر دسترسی شبکه بر اساس whitelist سخت‌گیرانه محدود شده باشد یا مسیر اتصال به مقصد کاملاً قطع شده باشد، این  
برنامه به‌تنهایی نمی‌تواند مشکل را برطرف کند.  
  
## قابلیت‌های اصلی  
  
* ایجاد پروکسی HTTP محلی روی ویندوز  
* resolve کردن دامنه‌ها با استفاده از DNS over HTTPS  
* پشتیبانی از چند DoH endpoint و استفاده از fallback در صورت خطا  
* امکان عبور دادن درخواست‌های DoH از یک پروکسی جداگانه  
* امکان عبور دادن اتصال‌های نهایی از یک upstream proxy  
* تنظیم خودکار پروکسی سیستم ویندوز  
* به‌روزرسانی فایل `hosts` ویندوز در صورت اجرا با دسترسی Administrator  
* ثبت دامنه‌های resolve شده در فایل `resolved.txt`  
* حالت `Optimize DNS` برای آزمایش، انتخاب و cache کردن IPهای مناسب‌تر  
* حالت `Emergency Mode` برای اولویت دادن به نتایج قبلی و فایل `hosts` در زمان اختلال DNS  
* نمایش لاگ‌های جزئی‌تر برای بررسی خطاها و وضعیت اتصال  
  
## طرز استفاده  
  
این راهنما بر اساس نسخه‌ی `exe` برنامه نوشته شده است. اگر برنامه را از سورس اجرا می‌کنید، بخش «اجرای نسخه‌ی سورس» را هم  
ببینید.  
  
## شروع سریع  
  
اگر فقط می‌خواهید برنامه را اجرا کنید و وارد جزئیات تنظیمات نشوید، مراحل زیر کافی است:  
  
1. فایل `exe` برنامه را اجرا کنید.  
2. در منوی برنامه با کلیدهای جهت‌دار بین گزینه‌ها جابه‌جا شوید.  
3. برای تغییر هر گزینه، آن را انتخاب کنید و مقدار جدید را وارد کنید؛ در بعضی گزینه‌ها می‌توانید با کلیدهای چپ و راست  
 مقدار را تغییر دهید.  
4. گزینه‌ی `Start` را انتخاب کنید و کلید Enter را بزنید تا پروکسی اجرا شود.  
  
در استفاده‌ی معمول، کافی است فایل `doh-list.txt` شامل endpointهای معتبر DoH باشد و برنامه را اجرا کنید.  
  
## گزینه‌های منوی برنامه  
  
| گزینه | توضیح |  
|--------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------|  
| `listen IP`                          | آدرسی که برنامه روی آن گوش می‌دهد. برای استفاده‌ی معمول، مقدار پیش‌فرض کافی است.                                                                   |  
| `listen port`                        | پورتی که پروکسی محلی روی آن اجرا می‌شود. اگر برنامه‌ی دیگری از این پورت استفاده می‌کند، آن را تغییر دهید.                                          |  
| `Set System Proxy`                   | اگر روشن باشد، پروکسی ویندوز به‌صورت خودکار روی این برنامه تنظیم می‌شود.                                                                           |  
| `DoH File`                           | فایل شامل آدرس‌های DoH endpoint که برنامه برای resolve کردن دامنه‌ها از آن استفاده می‌کند.                                                         |  
| `DoH Proxy Address`                  | آدرس پروکسی‌ای که خودِ درخواست‌های DoH از آن عبور می‌کنند.                                                                                         |  
| `Use DoH Proxy`                      | استفاده یا عدم استفاده از مقدار `DoH Proxy Address`.                                                                                               |  
| `Use Upstream Proxy`                 | اگر روشن باشد، اتصال‌های نهایی به مقصد از یک پروکسی خروجی دیگر عبور می‌کنند.                                                                       |  
| `Upstream Proxy Address`             | آدرس پروکسی خروجی برای اتصال‌های نهایی.                                                                                                            |  
| `Output Resolved Hosts`              | فایلی که دامنه‌های resolve شده و IPهای آن‌ها در آن ذخیره می‌شوند.                                                                                  |  
| `Auto Change Hosts (requires admin)` | اگر روشن باشد، برنامه فایل `hosts` ویندوز را به‌روزرسانی می‌کند. این گزینه معمولاً به دسترسی Administrator نیاز دارد.                              |  
| `Optimize DNS`                       | IPهای مناسب‌تر را آزمایش و cache می‌کند تا اتصال پایدارتر یا سریع‌تر انجام شود.                                                                    |  
| `Emergency Mode`                     | ابتدا از نتایج قبلی و فایل `hosts` استفاده می‌کند و فقط در صورت نیاز سراغ DoH می‌رود. این حالت برای زمان‌هایی مناسب است که DNS شبکه در دسترس نیست. |  
| `Verbose`                            | جزئیات بیشتری از روند اتصال، resolve و fallbackها نمایش می‌دهد.                                                                                    |  
| `Start`                              | برنامه را با تنظیمات فعلی اجرا می‌کند.                                                                                                             |  
  
## اجرای نسخه‌ی آماده‌ی `exe`  
  
* فایل `exe` برنامه را اجرا کنید.  
* اگر فایل `run_proxy.bat` همراه بسته‌ی انتشار وجود دارد، می‌توانید برنامه را از طریق آن هم اجرا کنید.  
* اگر برنامه را بدون آرگومان اجرا کنید، منوی تعاملی باز می‌شود و می‌توانید تنظیمات را همان‌جا تغییر دهید.  
  
## اجرای نسخه‌ی سورس  
  
فقط در صورتی که می‌خواهید برنامه را از سورس اجرا کنید، به Python و وابستگی‌های زیر نیاز دارید:  
<div dir="ltr" align="left">
  
```powershell
pip install requests dnspython
```
</div>
  
سپس می‌توانید برنامه را اجرا کنید:  
<div dir="ltr" align="left">
  
```powershell  
python doh_http_proxy.py  
```  
</div>
  
## فایل DoH  
  
فهرست DoH endpointها در فایل `doh-list.txt` قرار می‌گیرد. هر خط باید یک URL معتبر باشد. خط‌های خالی و خط‌هایی که با `#`  
شروع می‌شوند نادیده گرفته می‌شوند.  
  
نمونه:  
<div dir="ltr" align="left">
  
```txt  
https://cloudflare-dns.com/dns-query  
https://dns.google/dns-query  
https://security.cloudflare-dns.com/dns-query  
```  
</div>

اولین آدرس به‌عنوان DoH اصلی استفاده می‌شود و آدرس‌های بعدی در صورت خطا یا عدم پاسخ‌گویی، به‌عنوان fallback امتحان  
می‌شوند.  
  
## اجرا با آرگومان‌های خط فرمان  
  
اگر برنامه را با آرگومان اجرا کنید، مقدارهای داده‌شده فقط برای همان اجرا اعمال می‌شوند و نسبت به تنظیمات ذخیره‌شده  
اولویت دارند.  
  
نمونه در نسخه‌ی `exe`:  
  
<div dir="ltr" align="left">
  
```powershell  
doh_http_proxy.exe --listen 127.0.0.1 --port 8080 --verbose  
```  
</div>
نمونه در نسخه‌ی سورس:  
<div dir="ltr" align="left">
  
```powershell  
python doh_http_proxy.py --listen 127.0.0.1 --port 8080 --verbose  
```  
</div>  

## تنظیم پروکسی در سیستم یا برنامه‌ها  
  
آدرس پیش‌فرض گوش‌دادن برنامه به‌صورت زیر است:  
  
<div dir="ltr" align="left">
  
```txt  
0.0.0.0:8080  
```  
</div>    

اگر گزینه‌ی `Set System Proxy` روشن باشد، پروکسی سیستم ویندوز به‌صورت خودکار روی برنامه تنظیم می‌شود.  
  
اگر می‌خواهید فقط یک برنامه‌ی خاص از این پروکسی استفاده کند، تنظیمات پروکسی همان برنامه را روی یکی از آدرس‌های زیر قرار  
دهید:  
  
<div dir="ltr" align="left">

```txt  
http://127.0.0.1:8080  
```  
</div>   

یا اگر مقدار `listen IP` و `listen port` را تغییر داده‌اید، از آدرس و پورت انتخابی خودتان استفاده کنید.  
  
## فایل تنظیمات ذخیره‌شده  
  
برنامه تنظیمات آخرین اجرا را به‌صورت خودکار در مسیر زیر ذخیره می‌کند:  
  
<div dir="ltr" align="left">

```txt  
%USERPROFILE%\.doh_http_proxy\startup_config.json  
```  
</div>   

در اجرای بعدی، این تنظیمات بارگذاری می‌شوند. اگر هنگام اجرا آرگومان خط فرمان وارد کنید، مقدارهای خط فرمان روی تنظیمات  
ذخیره‌شده اولویت خواهند داشت.  
  
## تنظیمات  
  
### تنظیمات اصلی  
  
| گزینه | توضیح | مقدار پیش‌فرض |  
|----------------------|--------------------------------------------------------------------------------------------------------------|----------------|  
| `listen`             | آدرس گوش‌دادن پروکسی | `0.0.0.0`      |  
| `port`               | پورت پروکسی محلی | `8080`         |  
| `doh_file`           | فایل شامل فهرست DoH endpointها | `doh-list.txt` |  
| `doh_method`         | متد درخواست DoH. مقدارهای مجاز: `GET` یا `POST`                                                              | `POST`         |  
| `doh_proxy`          | پروکسی اختیاری برای خودِ درخواست‌های DoH، مانند `http://127.0.0.1:7890`                                      | —              |  
| `use_doh_proxy`      | فعال یا غیرفعال کردن استفاده از `doh_proxy`                                                                  | فعال |  
| `use_upstream_proxy` | فعال یا غیرفعال کردن پروکسی برای اتصال کلاینت‌ها به مقصد | غیرفعال |  
| `upstream_proxy`     | پروکسی اختیاری برای اتصال‌های خروجی کلاینت‌ها. اگر `use_upstream_proxy` روشن باشد، این مقدار باید تنظیم شود. | —              |  
| `output`             | فایل خروجی برای ثبت دامنه‌های resolve شده به شکل `[IP] [domain]`                                             | `resolved.txt` |  
  
### رفتار شبکه و امنیت  
  
| گزینه | توضیح | مقدار پیش‌فرض |  
|---------------------|----------------------------------------------------------------------------------------------------|---------------|  
| `set_system_proxy`  | تنظیم خودکار پروکسی سیستم ویندوز | فعال |  
| `auto_change_hosts` | به‌روزرسانی فایل `hosts` ویندوز. این قابلیت معمولاً به دسترسی Administrator نیاز دارد.             | فعال |  
| `insecure_doh_tls`  | غیرفعال کردن بررسی TLS برای endpointهای DoH. فقط در شرایط خاص و با احتیاط استفاده شود.             | غیرفعال |  
| `family`            | تعیین خانواده IP برای resolve. مقدارهای مجاز: `ipv4`، `ipv6`، `both-ipv4-first`، `both-ipv6-first` | `ipv4`        |  
| `client_timeout`    | timeout خواندن اولیه از کلاینت | `10`          |  
| `connect_timeout`   | timeout اتصال به مقصد یا پروکسی | `10`          |  
| `idle_timeout`      | timeout بیکاری tunnel                                                                              | `300`         |  
| `max_header_bytes`  | حداکثر اندازه‌ی مجاز هدر HTTP                                                                      | `65536`       |  
  
### بهینه‌سازی و گزارش  
  
| گزینه | توضیح | مقدار پیش‌فرض |  
|---------------------------------|------------------------------------------------------------------------------|-----------------------------|  
| `optimize_dns`                  | فعال یا غیرفعال کردن cache و بهینه‌سازی اتصال بر اساس IPهای بهتر | فعال |  
| `optimize_dns_refresh_interval` | فاصله‌ی زمانی sweep پس‌زمینه برای به‌روزرسانی Optimize DNS                   | `60` ثانیه |  
| `optimize_dns_report`           | فایل گزارش JSON برای نتایج Optimize DNS                                      | `optimized_dns_report.json` |  
| `emergency_mode`                | قبل از DoH، ابتدا `optimized_dns_report.json` و فایل `hosts` را بررسی می‌کند | غیرفعال |  
  
### ثبت لاگ و اشکال‌زدایی  
  
| گزینه | توضیح | مقدار پیش‌فرض |  
|-----------|------------------------------------------------|---------------|  
| `min_ttl` | حداقل TTL cache                                | `30`          |  
| `max_ttl` | حداکثر TTL cache                               | `300`         |  
| `verbose` | چاپ جزئیات اتصال، DoH و fallbackها در `stderr` | غیرفعال |  
  
## نمونه‌های استفاده  
  
اجرای نسخه‌ی `exe` روی آدرس و پورت مشخص همراه با لاگ کامل:  

<div dir="ltr" align="left">

```powershell  
doh_http_proxy.exe --listen 127.0.0.1 --port 8080 --verbose  
```  
</div> 
  
اجرای نسخه‌ی سورس با همان تنظیمات:  
  
<div dir="ltr" align="left">

```powershell  
python doh_http_proxy.py --listen 127.0.0.1 --port 8080 --verbose  
```  
</div> 
  
اجرای برنامه بدون تنظیم خودکار پروکسی سیستم و بدون تغییر فایل `hosts`:  
  
<div dir="ltr" align="left">

```powershell  
python doh_http_proxy.py --no-set-system-proxy --no-auto-change-hosts  
```  
</div> 
  
استفاده از upstream proxy برای اتصال‌های خروجی:  
  
<div dir="ltr" align="left">

```powershell  
python doh_http_proxy.py --use-upstream-proxy --upstream-proxy http://127.0.0.1:7890  
```  
</div> 
  
## نکات مهم  
  
* برای تغییر فایل `hosts` باید برنامه را با دسترسی Administrator اجرا کنید.  
* اگر پروکسی سیستم را فعال می‌کنید، پس از پایان کار مطمئن شوید تنظیمات پروکسی ویندوز به وضعیت دلخواه شما برگشته است.  
* اگر DoH endpoint اصلی پاسخ ندهد، برنامه endpointهای بعدی فایل `doh-list.txt` را امتحان می‌کند.  
* در حالت `Emergency Mode`، کیفیت نتایج قبلی اهمیت زیادی دارد؛ بنابراین بهتر است قبل از استفاده از این حالت، برنامه  
 یک‌بار در شرایط عادی اجرا شده باشد تا داده‌های معتبر در cache و گزارش‌ها ذخیره شوند.  
* گزینه‌ی `insecure_doh_tls` بررسی TLS را غیرفعال می‌کند و فقط باید در شرایط خاص و با آگاهی از ریسک‌های امنیتی استفاده  
 شود.
